# Tracking and Amplifying Linguistic Ambiguity in LLM-Generated Connections Puzzles

Code, data, and paper for a framework that generates NYT *Connections*-style puzzles
with controlled, amplified linguistic ambiguity — and, critically, **validates the
difficulty metric used to evaluate them against ground truth** instead of assuming it.

Every number, table, and figure in the paper regenerates from this repository.

## Headline results (all reproducible below)

- **Difficulty-metric validation** — on 422 official NYT puzzles (1,688 groups with
  ground-truth difficulty levels 0–3), intra-group embedding similarity correlates with
  official difficulty at **Spearman ρ = −0.449** (p < 1e−83), decreases monotonically
  per level (0.349 → 0.324 → 0.292 → 0.251), and orders within-puzzle group pairs
  correctly **70%** of the time (chance 50%).
- **Ambiguity ladder realized** — across the pipeline's four tiers, mean WordNet sense
  counts rise 6.1 → 16.4 → 14.9 → 25.0 and idiomatic multi-word phrases rise 0% → 50%,
  while real NYT groups sit at 7.4 senses regardless of official level.
- **Harder groups** — pipeline groups score 0.275 ± 0.126 mean intra-group similarity
  vs 0.304 ± 0.093 for real NYT groups (Mann-Whitney p = 1e−5) and 0.299 ± 0.097 for
  single-prompt GPT-4o generation.

│   ├── rcot_system.txt, rcot_template.txt

│   ├── compare_datasets.py       # real vs synthetic vs pipeline distributions
│   └── make_figures.py           # regenerates every paper figure
├── experiments/

**Dataset provenance:** real puzzles come from the community-maintained
[NYT-Connections-Answers](https://github.com/Eyefyre/NYT-Connections-Answers) archive
(puzzles from 2023-06-12 onward), used for non-commercial research.

## Reproduce every number in the paper

```bash
pip install -r requirements.txt
python -c "import nltk; nltk.download('wordnet')"

# Table 1 / Figure 2 — metric validation against official NYT levels
python metrics/validate_difficulty.py

# Table 2 / Figure 3 — ambiguity ladder over the 50-puzzle pipeline corpus
python metrics/ambiguity_metrics.py

# Table 3 / Figure 4 — real vs synthetic vs pipeline difficulty
python metrics/compare_datasets.py

# regenerate all paper figures from the results/ JSONs
python metrics/make_figures.py
```

The first embedding run downloads `all-mpnet-base-v2` (~420 MB) from Hugging Face.

## Generate new puzzles

```bash
export OPENAI_API_KEY=sk-...
python pipeline/generate_puzzles.py --num-puzzles 50 --model gpt-4o --out-dir data/generated
```

Each puzzle writes three JSON artifacts (stage 1 generation trace, stage 2 WordNet
verification, stage 3 RCoT audit).

## Evaluate solvers

```bash
export OPENAI_API_KEY=sk-...
# real NYT puzzles, CoT prompting
python experiments/solve_puzzles.py --dataset real --model gpt-4o --strategy cot --limit 100
# generated puzzles, ToT prompting
python experiments/solve_puzzles.py --dataset generated --file Testing_Data.json --strategy tot
```

Reports puzzle solve rate, group accuracy, and per-official-level accuracy, with full
per-instance logs under `results/solver/`. The paper deliberately reports no solver
numbers until this harness has been run end-to-end (see REVIEWER_RESPONSE.md).
